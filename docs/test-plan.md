## Affected paths (skill execution branches)

- `SKILL.md` — opening utterance, 7 interview Qs, classifier prompt invocation, build loop, deploy walkthrough, recap generation, closing
- `template-picker.md` — classifier prompt (force-fit logic)
- `templates/internal-tool/` — static HTML + localStorage CRUD
- `templates/dashboard/` — static HTML + Chart.js + hardcoded data.json
- `templates/prototype/` — static multi-page clickable HTML
- `recap.md.tmpl` — exit memo
- `setup` — symlink installer

## Dogfood tests (mandatory before public launch — Sunday morning)

Run `/first-build` end-to-end yourself once per template:

1. **internal-tool dogfood:** Pretend you're a PM who wants a personal-task tracker. Answer the 7 interview questions. Confirm classifier picks `internal-tool`. Confirm `git push` + Pages toggle produce a live URL within 5 minutes. Verify the deployed tool actually works (add a row, refresh page, row persists via localStorage).
2. **dashboard dogfood:** Pretend you're a PM who wants a metrics dashboard. Confirm classifier picks `dashboard`. Confirm Chart.js renders with the pre-baked data shape. Verify chart is readable.
3. **prototype dogfood:** Pretend you're a PM who wants a 3-page clickable mockup. Confirm classifier picks `prototype`. Verify all 3 pages link correctly.

**Pass criteria:** All 3 templates produce a live `https://<user>.github.io/<repo>/` URL within 30 minutes of running `/first-build`. Each deployed result is share-worthy (passes the "would you post this in Slack?" sniff test).

## Manual / interaction tests

- **Override flow:** Run `/first-build`, deliberately phrase the interview so the classifier picks the wrong template, then type "override <other-template>". Verify the skill rewrites `spec.md`'s `template` field, re-runs the bending instructions for the new template, and proceeds. Build should still complete in time-budget.
- **gh CLI missing:** Temporarily move `gh` out of PATH. Verify the skill falls back gracefully to web UI instructions for repo creation.
- **No GitHub account:** Test the "no account" precheck path — skill should print signup link and wait for PM to come back with a username.
- **Repo name conflict:** Pre-create a repo with the name the skill is about to pick. Verify skill suggests `<name>-v2` instead of failing.
- **Long Pages propagation:** If Pages takes >5 min on a new repo, verify the skill's manual paste-back instruction is clear ("refresh in your browser; paste URL when live"). PM should not be confused.

## Recap reusability test (Wednesday-Friday next week)

This is the V1-validates-V2-direction signal:

- Pick 1 of the first 3 DM'd PMs.
- Ask them to attempt a **second** build using **only** their first build's `recap.md` — no `/first-build` invocation, no rails.
- Observe whether the recap was sufficient. Did the PM ship a second tool? If yes, the recap works as designed. If not, V2 should be Approach B (laddered `/next-build` skill) or C (glass box).

## Edge cases for V1

- PM types a spec that doesn't fit any template cleanly → force-fit picks closest + explains bend honestly. Verify PM doesn't dead-end into "no template fits."
- PM gives a one-word answer to an interview question → skill should ask one short follow-up, not bail.
- PM hits Min 30 without a live URL (Pages still propagating) → recap is already at `recap.md`; skill handoff still works.
- Browser-only static templates fail to load due to GitHub Pages quirk (e.g., case-sensitive paths, missing `index.html`) → covered by per-template dogfood test.
